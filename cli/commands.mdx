### Authentication

Before using xCommand, you need to authenticate. You can trigger the login process by pressing `L` when the CLI is running in interactive mode.

<Note>
  To run xCommand in server mode, you will need xCommand API Token as `XCOMMAND_API_KEY` environment variable. You can get it from your xCommand.ai/platform once logged in.
</Note>

## Root Commands

Root commands are executed directly from your terminal and govern the CLI's behavior and environment.

### Help and Version

- **Help**: Show available commands and options.
  ```bash
  xcommand help
  ```
- **Version**: Show the current CLI version.
  ```bash
  xcommand version
  ```

### Session Management

- **List Sessions**: View all local sessions, sorted by most recent first.
  ```bash
  xcommand sessions
  ```
- **Resume Session**: Rejoin a specific session by its ID.
  ```bash
  xcommand [session-id]
  ```

### Environment and Lifecycle

- **Update**: Check for and install the latest CLI version.
  ```bash
  xcommand update
  ```
- **Server Mode**: Start xCommand as a background proxy (defaults to port `8080`).
  ```bash
  xcommand server
  ```
  Use the `--port` flag for a custom port:
  ```bash
  xcommand server --port 9000
  ```

## CLI Interface

Once logged in, you can interact with the xCommand directly from your terminal. 

### Slash Commands

While in the interactive chat, you can use slash commands to perform specific actions:

- `/knowledge`: Manage your knowledge base.
  - `/knowledge list`: List installed modules.
  - `/knowledge search [query]`: Search for modules.
  - `/knowledge add <name>`: Install a module.
  - `/knowledge delete <name>`: Remove a module.
- `/plugin`: Manage your plugins.
  - `/plugin slack connect`: Connect xCommand to your Slack workspace.
- `/insights`: View session and token usage insights.
- `/docs`: View documentation.
- `/platform`: View platform to manage xCommand account.
- `/logout`: Sign out of your account.
- `/bug <description>`: Report a bug to the xCommand team.
- `/clear`: Clear the chat history.
- `/license`: View the xCommand limited license agreement.
- `/copy`: Copy the last AI response to your system clipboard.
- `/mouse on|off`: Toggle mouse interaction. Useful if you want to select text naturally (with `/mouse off`) or use the scroll wheel (with `/mouse on`).
- `/brief [prompt]`: Initialise the conversation from your local memories or set a custom briefing prompt. 
  - Typing `/brief` (no arguments) triggers a proactive summary of your ongoing work or any relevant context found in your memory store.
  - Typing `/brief [custom instructions]` saves those instructions locally (`~/.xcommand/brief_prompt.md`). xCommand will prioritize your custom prompt during all future session briefings.
  - Typing `/brief clear` (or `reset` or `default`) removes your custom prompt and reverts to the standard xCommand briefing.
- `/help`: Show the help screen.

<Tip>
  When mouse interaction is enabled (`/mouse on`), you can still select and copy text by holding the **Option** key (macOS) or **Shift** key (Linux/Windows) while clicking and dragging.
</Tip>

### Shell Mode

You can run local shell commands directly from the xCommand interface by prefixing them with a backslash (`\`):

```bash
\ls -la
```

```bash
\cd
```

### Session Management

When you exit a session, xCommand saves it. You can rejoin a saved session by running:

```bash
xcommand [session-id]
```

### Update xCommand

To update xCommand to the latest version, run:

```bash
xcommand update
```
