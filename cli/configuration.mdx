xCommand CLI can be configured using a `config.json` file or via command-line arguments. This allows you to persist settings like your API key, preferred server port, and execution mode.

## Configuration File

By default, xCommand looks for a configuration file at:
`~/.xcommand/config.json`

### File Structure

The configuration file is a simple JSON object. Here is the default structure:

```json
{
  "api_key": "your-api-key-here",
  "server_port": 8080,
  "mode": "safe",
  "github_token": "your-github-token-here",
  "gcloud_service_account": "your-sa@project.iam.gserviceaccount.com",
  "gcloud_service_account_json_path": "/path/to/key.json",
  "aws_profile": "default",
  "aws_access_key_id": "your_access_key_id",
  "aws_secret_access_key": "your_secret_access_key",
  "aws_region": "us-east-1",
  "azure_client_id": "your_client_id",
  "azure_client_secret": "your_client_secret",
  "azure_tenant_id": "your_tenant_id",
  "azure_subscription_id": "your_subscription_id",
  "do_access_token": "your_digitalocean_access_token",
  "linear_token": "your_linear_api_token",
  "notion_token": "your_notion_api_token"
}
```

### Fields

| Field | Description | Default |
| :--- | :--- | :--- |
| `api_key` | Your xCommand API key for authentication. | `None` |
| `server_port` | The port used when running in [server mode](/cli/commands#server-mode). | `8080` |
| `mode` | The default execution mode (`safe` or `standard`). | `safe` |
| `github_token` | A GitHub Personal Access Token for the [GitHub Integration](/cli/integrations/github). | `None` |
| `gcloud_service_account` | Target Google Cloud Service Account for the [GCP Integration](/cli/integrations/google-cloud). | `None` |
| `gcloud_service_account_json_path` | Path to a Service Account JSON key for the [GCP Integration](/cli/integrations/google-cloud). | `None` |
| `aws_profile` | Target AWS Profile for the [AWS Integration](/cli/integrations/aws). | `None` |
| `aws_access_key_id` | AWS Access Key ID for the [AWS Integration](/cli/integrations/aws). | `None` |
| `aws_secret_access_key` | AWS Secret Access Key for the [AWS Integration](/cli/integrations/aws). | `None` |
| `aws_region` | AWS Region for the [AWS Integration](/cli/integrations/aws). | `None` |
| `azure_client_id` | Azure Client ID for the [Azure Integration](/cli/integrations/azure). | `None` |
| `azure_client_secret` | Azure Client Secret for the [Azure Integration](/cli/integrations/azure). | `None` |
| `azure_tenant_id` | Azure Tenant ID for the [Azure Integration](/cli/integrations/azure). | `None` |
| `azure_subscription_id` | Azure Subscription ID for the [Azure Integration](/cli/integrations/azure). | `None` |
| `do_access_token` | DigitalOcean Personal Access Token for the [DigitalOcean Integration](/cli/integrations/digital-ocean). | `None` |
| `linear_token` | Linear Personal API Key for the [Linear Integration](/cli/integrations/linear). | `None` |
| `notion_token` | Notion Personal Access Token | `None` |

## CLI Arguments

You can override the default configuration using command-line flags.

### Custom Configuration Path

If you want to use a specific configuration file instead of the default one, use the `--config-file` flag:

```bash
xcommand --config-file ./custom-config.json
```

### Server Port Override

When starting the server, you can specify a port that takes precedence over the configuration file:

```bash
xcommand server --port 9000
```

## Environment Variables

For automated environments or CI/CD pipelines, you can use the `XCOMMAND_API_KEY` environment variable to authenticate without a configuration file.

```bash
export XCOMMAND_API_KEY=your_secret_token
xcommand bypass-login
```
